--------------------------------------------------------------------------------
1. ЗАДАЧА: СРОЧНЫЙ БАГФИКС HYDRATION (app/page.tsx)
--------------------------------------------------------------------------------

Проблема (диагностика):
 У части пользователей страница отображалась, но кнопки не работали — типичный
  признак hydration mismatch: сервер и клиент отдавали разный HTML/React-дерево,
  гидратация не завершалась, интерактивность не подключалась.

Выполненные исправления:

  1) Таймер (Timer)
     - Было: useState(target - Date.now()) — разное время на SSR и клиенте.
     - Стало: diff начинается с 0; флаг mounted; в useEffect: setMounted(true),
       setDiff(target - Date.now()), setInterval на 1 с.
     - До mounted === true показывается статичный текст без цифр счётчика:
       «Вторник, 14 апреля 2026 — 19:00 МСК» в обёртке с классом nu-timer-row.

  2) Матричный фон (на тот момент — SVG)
     - Было: Math.random() при рендере колонок/символов — разный вывод SSR/CSR.
     - Стало: весь фиксированный фоновый блок с матрицей только при mounted
       в компоненте Landing (отдельный useState mounted + useEffect setMounted(true)).

  3) Тема / scroll
     - Проверено: window только в useEffect (скролл, модалка). localStorage/matchMedia
       при рендере не использовались.

  4) Скан файла
     - Date.now() вне useEffect устранён (таймер).
     - Math.random() при SSR устранён (матрица за mounted).
     - window/document вне useEffect — не найдены.

  5) Подстраховка
     - На корневой div Landing добавлен suppressHydrationWarning.

Ограничения по запросу:
  Тексты, стили и общая структура страницы не менялись, кроме необходимого для
  безопасности гидратации.

Результат проверки:
  npm run build — успешно.

Риски (кратко):
  - Кратковременное отсутствие «матрицы» до mounted (ожидаемо).
  - Дублирование даты: заголовок секции + плейсхолдер таймера до гидратации.
  - suppressHydrationWarning не отменяет необходимость чинить реальные источники mismatch.


--------------------------------------------------------------------------------
2. ЗАДАЧА: МАТРИЧНЫЙ ФОН ЗАНОВО — CANVAS (app/page.tsx)
--------------------------------------------------------------------------------

Проблема:
  Фон отсутствовал или не отображался; требовалась «живая» матрица на canvas,
  не SVG/CSS background.

Выполнено:

  - import: добавлен useRef.
  - Landing: matrixRef = useRef<HTMLCanvasElement>(null).
  - Блок {mounted && (...)} заменён на:
      * слой свечений с radial-gradient и fallback цветов (C.bgGlow1/bgGlow2 или тёмная/светлая заглушка);
      * <canvas ref={matrixRef} /> с opacity 0.12 (dark) / 0.05 (light).
  - useEffect с зависимостями [mounted, theme]:
      * if (!mounted) return;
      * getContext("2d"), resize по window, addEventListener("resize"),
      * массив drops; при resize пересчёт columns и drops (исправление рассинхрона
        после изменения ширины окна — сверх ТЗ, но нужно для корректной работы);
      * цикл draw с requestAnimationFrame, отмена в cleanup через cancelAnimationFrame
        и removeEventListener("resize").
  - Набор символов: катакана + цифры, в исходнике записано через \uXXXX для стабильной UTF-8 в файле.
  - SVG-матрица и переменная matrixFill удалены.

Результат проверки:
  npm run build — успешно.

Риски (кратко):
  - Нагрузка на CPU/GPU из-за полного fillRect и множества fillText каждый кадр.
  - При смене темы эффект перезапускается — краткий визуальный сброс анимации.
  - Зависимость от системного шрифта monospace для отрисовки катаканы.


--------------------------------------------------------------------------------
3. САМОПРОВЕРКА №1 (после hydration-фикса, до/включая API)
--------------------------------------------------------------------------------

Гидратация:
  - Таймер и mounted-гейт для фона согласованы с SSR.
  - suppressHydrationWarning на корне — только подстраховка.

Секреты:
  - Клиент без токенов; Telegram — env на сервере в /api/lead.

Ошибки / граничные случаи:
  - Модалка: catch без детального лога; fetch при закрытии модалки теоретически может
    дать setState после размонтирования.
  - API: при catch и при отсутствии env возможен ответ { ok: true } — пользователь
    может думать, что заявка ушла, хотя в Telegram ничего не отправлено.

Репозиторий:
  - Дублирование page.tsx в корне и app/page.tsx — риск путаницы.

Объём:
  - app/page.tsx сильно больше 300 строк — техдолг по разбиению.


--------------------------------------------------------------------------------
4. САМОПРОВЕРКА №2 (после canvas-матрицы)
--------------------------------------------------------------------------------

Гидратация:
  - Canvas только при mounted; Date.now только в эффекте таймера; window в эффектах.

Матрица:
  - Cleanup rAF + resize корректен; при смене theme цикл пересоздаётся.
  - resize синхронизирует drops с шириной.

Производительность:
  - Плотная отрисовка каждый кадр — возможны просадки на слабых устройствах.

Прочее:
  - Корневой page.tsx по-прежнему отдельный файл — дублирование версий.
  - Файл страницы монолитный.
